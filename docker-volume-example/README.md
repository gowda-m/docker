# Docker Volume Example

This project demonstrates how to use Docker volumes for persistent storage with containers.

Docker volumes allow data generated by containers to be stored outside the container filesystem.  
This ensures data remains available even if the container is stopped or removed.

---

## Create Docker Volume

Create a Docker volume using the following command:

```bash
docker volume create mydata
```

List available Docker volumes:

```bash
docker volume ls
```

---

## Run Container with Volume

Run an nginx container and attach the created volume:

```bash
docker run -d -p 8081:80 -v mydata:/usr/share/nginx/html --name nginx-volume nginx
```

Explanation:

- `-d` → Run container in detached mode  
- `-p 8081:80` → Map host port **8081** to container port **80**  
- `-v mydata:/usr/share/nginx/html` → Mount Docker volume  
- `--name nginx-volume` → Assign container name
